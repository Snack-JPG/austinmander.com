## The Concept

Every frontend developer knows the pain: you build a new component, manually test it in a few scenarios, ship it, then discover edge cases in production. **SmokeRunner AI eliminates this cycle.**

It generates comprehensive smoke tests for your components, runs them against your localhost, and feeds failures back to an AI agent until everything passes. The fastest way to trust new code.

## How It Works

```bash
# Install and run on any component
npx smokerunner-ai ./src/components/Button.tsx

# SmokeRunner AI will:
# 1. Analyze your component's props and API
# 2. Generate Playwright tests for common scenarios
# 3. Run tests against your dev server
# 4. Fix failures by updating component or tests
# 5. Repeat until all tests pass
```

### The Testing Loop

**Step 1: Component Analysis**
SmokeRunner scans your component file to understand:

- Props interface and types
- Conditional rendering logic
- Event handlers and callbacks
- Accessibility requirements

**Step 2: Test Generation**
Based on the analysis, it generates tests for:

- Default rendering states
- All prop combinations
- Edge cases (empty strings, null values)
- User interactions (clicks, form inputs)
- Accessibility compliance

**Step 3: Self-Healing**
When tests fail, the AI agent:

- Analyzes the failure screenshot and logs
- Determines if it's a component bug or test issue
- Suggests fixes for either the component or test
- Applies fixes and re-runs the tests
